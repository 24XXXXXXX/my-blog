# Markdown入门教程

Markdown是一种轻量级标记语言，让你可以使用易读易写的纯文本格式编写文档，然后转换成HTML。本教程将带你从零开始掌握Markdown的基础语法。

<!--more-->

## 一、什么是Markdown？

Markdown由John Gruber在2004年创建，其设计理念是：

- **易读性** - 即使是纯文本也具有良好的可读性
- **易写性** - 使用简单的标记符号即可完成排版
- **可转换** - 可以轻松转换为HTML、PDF等格式

### 为什么要学习Markdown？

1. **简单高效** - 专注内容创作，无需关心复杂的排版
2. **跨平台** - 纯文本格式，任何编辑器都能打开
3. **广泛应用** - GitHub、博客、文档、笔记等场景
4. **版本控制友好** - 纯文本便于Git管理

## 二、Front Matter详解

在Hugo等静态网站生成器中，文章开头的元数据部分称为Front Matter。

### `---` vs `+++` 的区别

Hugo支持三种Front Matter格式：

#### 1. YAML格式（使用 `---`）

```markdown
---
title: "文章标题"
date: 2025-11-11T13:44:18+08:00
draft: false
tags: ["标签1", "标签2"]
---
```

**特点：**
- 使用三个短横线 `---` 包围
- 语法简洁，可读性强
- 支持列表、字典等复杂结构
- **最常用的格式**

#### 2. TOML格式（使用 `+++`）

```markdown
+++
title = "文章标题"
date = 2025-11-11T13:44:18+08:00
draft = false
tags = ["标签1", "标签2"]
+++
```

**特点：**
- 使用三个加号 `+++` 包围
- 使用等号 `=` 赋值
- 语法类似配置文件
- Hugo的默认配置文件格式

#### 3. JSON格式（使用 `{` `}`）

```json
{
  "title": "文章标题",
  "date": "2025-11-11T13:44:18+08:00",
  "draft": false,
  "tags": ["标签1", "标签2"]
}
```

**特点：**
- 使用大括号包围
- 严格的JSON语法
- 较少使用

### 常用Front Matter字段

```yaml
---
title: "文章标题"              # 必需：文章标题
description: "文章描述"        # 推荐：SEO描述
date: 2025-11-11T13:44:18+08:00  # 必需：发布日期
lastmod: 2025-11-11T13:44:18+08:00  # 可选：最后修改日期
draft: false                   # 可选：是否为草稿（true不会发布）
categories:                    # 可选：分类
  - 技术
tags:                          # 可选：标签
  - Markdown
  - 教程
cover: "/images/cover.jpg"     # 可选：封面图片
---
```

## 三、基础语法

### 1. 标题

Markdown使用 `#` 号表示标题，支持6级标题：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

**效果：**

# 一级标题
## 二级标题
### 三级标题

**注意：** `#` 号后面要加空格！

### 2. 段落和换行

- **段落**：段落之间用空行分隔
- **换行**：行末加两个空格，或使用 `<br>`

```markdown
这是第一段。

这是第二段。

这是同一段的  
第二行（行末有两个空格）。
```

### 3. 文字样式

```markdown
**粗体文字**
*斜体文字*
***粗斜体***
~~删除线~~
`行内代码`
```

**效果：**

**粗体文字**  
*斜体文字*  
***粗斜体***  
~~删除线~~  
`行内代码`

### 4. 列表

#### 无序列表

使用 `-`、`+` 或 `*` 开头：

```markdown
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2
- 项目3
```

**效果：**

- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2
- 项目3

#### 有序列表

使用数字加点：

```markdown
1. 第一项
2. 第二项
3. 第三项
   1. 子项3.1
   2. 子项3.2
```

**效果：**

1. 第一项
2. 第二项
3. 第三项
   1. 子项3.1
   2. 子项3.2

### 5. 链接

```markdown
[链接文字](https://example.com)
[带标题的链接](https://example.com "鼠标悬停显示")
```

**效果：**

[链接文字](https://example.com)

### 6. 图片

```markdown
![图片描述](图片URL)
![带标题的图片](图片URL "图片标题")
```

**示例：**

```markdown
![Markdown Logo](https://markdown-here.com/img/icon256.png)
```

### 7. 引用

使用 `>` 符号：

```markdown
> 这是一段引用文字。
> 
> 可以有多行。
>> 嵌套引用
```

**效果：**

> 这是一段引用文字。
> 
> 可以有多行。
>> 嵌套引用

### 8. 代码块

#### 行内代码

使用反引号：

```markdown
这是 `行内代码` 示例。
```

#### 代码块

使用三个反引号，并指定语言：

````markdown
```javascript
function hello() {
  console.log("Hello, Markdown!");
}
```
````

**效果：**

```javascript
function hello() {
  console.log("Hello, Markdown!");
}
```

### 9. 分隔线

使用三个或以上的 `-`、`*` 或 `_`：

```markdown
---
***
___
```

**效果：**

---

### 10. 表格

```markdown
| 列1 | 列2 | 列3 |
| --- | --- | --- |
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

**效果：**

| 列1 | 列2 | 列3 |
| --- | --- | --- |
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |

**对齐方式：**

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :---: | ---: |
| 左 | 中 | 右 |
```

## 五、常见问题

### 1. 特殊字符如何显示？

如果要显示Markdown语法字符本身，使用反斜杠转义：

```markdown
\*这不是斜体\*
\#这不是标题
```

### 2. 如何在Hugo中隐藏部分内容？

使用 `<!--more-->` 标记，之前的内容会显示在文章列表中：

```markdown
这是摘要部分，会显示在首页。

<!--more-->

这是详细内容，只有点击进入文章才能看到。
```

### 3. 空格和缩进

Markdown会自动处理多余的空格，如果需要保留格式，使用代码块。
