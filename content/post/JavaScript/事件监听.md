掌握 addEventListener/removeEventListener、事件传播（捕获/冒泡）、以及事件委托，基本就能覆盖绝大多数前端交互需求。
<!--more-->

## 1. 事件监听的两种写法

### 方式1：DOM0（onxxx）

```js
const btn = document.querySelector("#btn");

btn.onclick = function () {
  console.log("clicked");
};
```

特点：

- 同一个事件类型只能绑定一个函数（后赋值会覆盖前一个）
- 不方便做捕获阶段、once、passive 等高级控制

### 方式2：DOM2（addEventListener，推荐）

```js
const btn = document.querySelector("#btn");

btn.addEventListener("click", () => {
  console.log("clicked");
});
```

特点：

- 同一个事件类型可以绑定多个监听器
- 支持捕获/冒泡阶段、once、passive、signal 等配置

## 2. removeEventListener 的正确姿势

`removeEventListener` 只能移除“同一个函数引用”。

错误示例：

```js
btn.addEventListener("click", () => console.log(1));
btn.removeEventListener("click", () => console.log(1));
// 无法移除，因为两个箭头函数不是同一个引用
```

正确示例：

```js
function handleClick() {
  console.log(1);
}

btn.addEventListener("click", handleClick);
btn.removeEventListener("click", handleClick);
```

## 3. 事件对象 Event（常用属性）

```js
btn.addEventListener("click", (event) => {
  console.log(event.type);      // click
  console.log(event.target);    // 真正触发事件的元素
  console.log(event.currentTarget); // 当前绑定监听器的元素
});
```

常用方法：

- `event.preventDefault()`：阻止默认行为（例如 a 跳转、form 提交）
- `event.stopPropagation()`：阻止继续传播（阻止冒泡/捕获继续向外传递）

## 4. 事件传播：捕获 -> 目标 -> 冒泡

事件传播顺序：

1. 捕获阶段（从外到内）
2. 目标阶段（到达目标元素）
3. 冒泡阶段（从内到外）

默认 `addEventListener("click", handler)` 监听的是 **冒泡阶段**。

监听捕获阶段：

```js
btn.addEventListener("click", handler, true);
// 或
btn.addEventListener("click", handler, { capture: true });
```

## 5. 事件委托（非常重要）

当你需要给大量子元素绑定同一个事件，或者子元素是动态生成的，优先用“事件委托”。

原理：利用事件冒泡，把监听器绑在父元素上，通过 `event.target` 判断点击的具体子元素。

```js
const list = document.querySelector("#list");

list.addEventListener("click", (e) => {
  const item = e.target.closest(".item");
  if (!item) return;
  console.log("clicked item:", item.dataset.id);
});
```

好处：

- 只绑定一次监听器，性能更好
- 动态新增的 `.item` 也能响应

## 6. addEventListener 的常用 options

```js
btn.addEventListener("click", handler, {
  capture: false,
  once: true,
  passive: false,
});
```

- `capture`：是否在捕获阶段触发
- `once`：触发一次后自动移除监听器
- `passive`：告诉浏览器回调不会调用 `preventDefault()`（滚动性能优化常用）

## 7. 常见事件类型

- 鼠标：`click`、`dblclick`、`mousedown`、`mouseup`、`mousemove`
- 键盘：`keydown`、`keyup`
- 表单：`input`、`change`、`submit`
- 焦点：`focus`、`blur`（注意：`focus/blur` 默认不冒泡，常用 `focusin/focusout`）

## 8. 常见坑点汇总

- **不要混淆 `target` 和 `currentTarget`**：
  - `target` 是“真正点到的元素”
  - `currentTarget` 是“你绑定监听器的元素”
- **removeEventListener 必须是同一个函数引用**
- **默认监听冒泡阶段**，需要捕获就传 `{ capture: true }`
- **委托时用 `closest` 更稳**：避免点到子节点（例如图标/文本）导致判断失败
