## 一、Linux 系统网络请求工具

### 1.1 curl 命令详解

curl（Client URL）是一个强大的命令行工具，用于传输数据。

#### 基本语法

```bash
curl [选项] [URL]
```

#### 基本使用

```bash
# 获取网页内容（输出到终端）
curl https://example.com
# curl: 命令名称
# https://example.com: 目标 URL

# 下载文件（保存为原文件名）
curl -O https://example.com/file.zip
# -O: 使用远程文件名保存（Output）
# 会保存为 file.zip

# 下载文件（指定文件名）
curl -o myfile.zip https://example.com/file.zip
# -o myfile.zip: 指定输出文件名（output）
# 会保存为 myfile.zip

# 显示下载进度
curl -# -O https://example.com/largefile.zip
# -#: 显示进度条（替代详细进度信息）

# 静默模式（不显示进度）
curl -s https://example.com
# -s: 静默模式（silent）
```

#### 常用选项详解

```bash
# -L: 跟随重定向
curl -L https://example.com/redirect
# -L: 如果服务器返回 301/302 重定向，自动跟随（Location）

# -I: 只获取 HTTP 头信息
curl -I https://example.com
# -I: 只显示响应头（HEAD 请求）

# -i: 显示响应头和内容
curl -i https://example.com
# -i: 包含响应头在输出中（include）

# -v: 详细模式（显示请求和响应详情）
curl -v https://example.com
# -v: 显示详细的通信过程（verbose）

# -X: 指定 HTTP 方法
curl -X POST https://api.example.com/data
# -X POST: 使用 POST 方法（默认是 GET）

# -H: 添加请求头
curl -H "Content-Type: application/json" https://api.example.com
# -H: 添加自定义 HTTP 头（Header）
# "Content-Type: application/json": 头部键值对

# -d: 发送 POST 数据
curl -X POST -d "name=John&age=25" https://api.example.com
# -d: 发送数据（data）
# "name=John&age=25": 表单数据

# -d 发送 JSON 数据
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"name":"John","age":25}' \
  https://api.example.com
# \: 行继续符（命令太长时换行）
# '{"name":"John","age":25}': JSON 格式数据

# --data-binary: 发送二进制数据
curl -X POST --data-binary @file.json https://api.example.com
# --data-binary: 发送原始二进制数据
# @file.json: 从文件读取数据（@ 表示文件）

# -u: HTTP 基本认证
curl -u username:password https://api.example.com
# -u: 用户认证（user）
# username:password: 用户名和密码

# -b: 发送 Cookie
curl -b "session=abc123" https://example.com
# -b: Cookie（browser cookie）
# "session=abc123": Cookie 键值对

# -c: 保存 Cookie
curl -c cookies.txt https://example.com
# -c: 保存 Cookie 到文件（cookie-jar）
# cookies.txt: Cookie 存储文件

# -A: 设置 User-Agent
curl -A "Mozilla/5.0" https://example.com
# -A: 设置用户代理（Agent）
# "Mozilla/5.0": 浏览器标识

# -e: 设置 Referer
curl -e "https://google.com" https://example.com
# -e: 设置来源页面（referer）
# "https://google.com": 引用页面 URL

# -x: 使用代理
curl -x http://proxy:8080 https://example.com
# -x: 代理服务器（proxy）
# http://proxy:8080: 代理地址和端口

# --limit-rate: 限制下载速度
curl --limit-rate 100K -O https://example.com/file.zip
# --limit-rate 100K: 限制速度为 100KB/s
# K: 千字节，M: 兆字节

# -C: 断点续传
curl -C - -O https://example.com/largefile.zip
# -C -: 自动检测断点位置继续下载（Continue）
# -: 自动确定偏移量

# -k: 忽略 SSL 证书验证
curl -k https://self-signed.example.com
# -k: 不验证 SSL 证书（insecure）
# 注意：不安全，仅用于测试

# --connect-timeout: 连接超时
curl --connect-timeout 10 https://example.com
# --connect-timeout 10: 连接超时 10 秒

# --max-time: 总超时时间
curl --max-time 60 https://example.com
# --max-time 60: 整个操作最多 60 秒

# -w: 输出格式化信息
curl -w "\nHTTP Code: %{http_code}\n" https://example.com
# -w: 输出自定义格式（write-out）
# %{http_code}: HTTP 状态码变量
```

#### 实用示例

```bash
# 下载文件并显示进度
curl -# -L -O https://github.com/user/repo/releases/download/v1.0/file.tar.gz
# -#: 显示进度条
# -L: 跟随重定向
# -O: 使用原文件名保存

# 测试 API 接口
curl -X GET \
  -H "Authorization: Bearer token123" \
  -H "Accept: application/json" \
  https://api.example.com/users
# -X GET: GET 请求
# -H "Authorization: Bearer token123": 认证令牌
# -H "Accept: application/json": 接受 JSON 响应

# POST 提交表单
curl -X POST \
  -F "file=@document.pdf" \
  -F "description=My Document" \
  https://api.example.com/upload
# -F: 表单数据（form）
# file=@document.pdf: 上传文件
# description=My Document: 文本字段

# 下载并解压
curl -L https://example.com/archive.tar.gz | tar xz
# |: 管道，将 curl 输出传给 tar
# tar xz: 解压 tar.gz 文件

# 检查网站响应时间
curl -w "\nTime: %{time_total}s\n" -o /dev/null -s https://example.com
# -w "\nTime: %{time_total}s\n": 输出总时间
# -o /dev/null: 丢弃输出内容
# -s: 静默模式

# 批量下载
for i in {1..10}; do
  curl -O https://example.com/file${i}.zip
done
# {1..10}: 生成 1 到 10 的序列
# ${i}: 变量替换
```

### 1.2 wget 命令详解

wget（Web Get）是一个专门用于下载文件的命令行工具。

#### 基本语法

```bash
wget [选项] [URL]
```

#### 基本使用

```bash
# 下载文件
wget https://example.com/file.zip
# wget: 命令名称
# https://example.com/file.zip: 目标 URL
# 自动保存为 file.zip

# 下载并指定文件名
wget -O myfile.zip https://example.com/file.zip
# -O myfile.zip: 指定输出文件名（Output document）

# 后台下载
wget -b https://example.com/largefile.zip
# -b: 后台下载（background）
# 下载日志保存在 wget-log

# 断点续传
wget -c https://example.com/largefile.zip
# -c: 继续下载未完成的文件（continue）

# 限制下载速度
wget --limit-rate=200k https://example.com/file.zip
# --limit-rate=200k: 限制速度为 200KB/s
```

#### 常用选项详解

```bash
# -q: 安静模式
wget -q https://example.com/file.zip
# -q: 不输出任何信息（quiet）

# -v: 详细模式（默认）
wget -v https://example.com/file.zip
# -v: 显示详细信息（verbose）

# -P: 指定下载目录
wget -P /tmp/downloads https://example.com/file.zip
# -P /tmp/downloads: 保存到指定目录（directory-prefix）

# -t: 重试次数
wget -t 5 https://example.com/file.zip
# -t 5: 失败后重试 5 次（tries）
# -t 0: 无限重试

# -T: 超时时间
wget -T 30 https://example.com/file.zip
# -T 30: 超时 30 秒（timeout）

# --spider: 只检查文件是否存在
wget --spider https://example.com/file.zip
# --spider: 不下载，只检查（像蜘蛛一样爬取）

# -i: 从文件读取 URL 列表
wget -i urls.txt
# -i urls.txt: 从文件读取 URL（input-file）
# urls.txt 每行一个 URL

# -r: 递归下载
wget -r https://example.com/directory/
# -r: 递归下载整个目录（recursive）

# -l: 递归深度
wget -r -l 2 https://example.com/
# -l 2: 递归深度为 2 层（level）

# -np: 不下载父目录
wget -r -np https://example.com/docs/
# -np: 不上溯到父目录（no-parent）

# -A: 只下载指定类型文件
wget -r -A "*.pdf" https://example.com/docs/
# -A "*.pdf": 只接受 PDF 文件（accept）

# -R: 拒绝指定类型文件
wget -r -R "*.html" https://example.com/docs/
# -R "*.html": 拒绝 HTML 文件（reject）

# --user: HTTP 认证用户名
wget --user=username --password=pass https://example.com/file.zip
# --user=username: 用户名
# --password=pass: 密码

# --header: 添加请求头
wget --header="Authorization: Bearer token" https://api.example.com/file
# --header="...": 添加 HTTP 头

# --user-agent: 设置 User-Agent
wget --user-agent="Mozilla/5.0" https://example.com/file.zip
# --user-agent="...": 设置用户代理

# --referer: 设置 Referer
wget --referer="https://google.com" https://example.com/file.zip
# --referer="...": 设置引用页面

# -e: 使用代理
wget -e use_proxy=yes -e http_proxy=proxy:8080 https://example.com
# -e: 执行命令（execute）
# use_proxy=yes: 启用代理
# http_proxy=proxy:8080: 代理地址

# --no-check-certificate: 忽略 SSL 证书
wget --no-check-certificate https://self-signed.example.com/file.zip
# --no-check-certificate: 不验证 SSL 证书
```

#### 实用示例

```bash
# 下载并重命名
wget -O latest.zip https://github.com/user/repo/archive/main.zip
# -O latest.zip: 保存为 latest.zip

# 后台下载大文件
wget -b -c https://example.com/ubuntu-22.04.iso
# -b: 后台下载
# -c: 支持断点续传

# 批量下载
wget -i download_list.txt -P /downloads
# -i download_list.txt: 从文件读取 URL
# -P /downloads: 保存到 /downloads 目录

# 镜像整个网站
wget -m -p -k -np https://example.com
# -m: 镜像（mirror，等同于 -r -N -l inf --no-remove-listing）
# -p: 下载所有必需文件（page-requisites）
# -k: 转换链接为本地链接（convert-links）
# -np: 不上溯到父目录

# 下载特定类型文件
wget -r -l 1 -A "*.pdf,*.doc" https://example.com/documents/
# -r: 递归
# -l 1: 深度 1
# -A "*.pdf,*.doc": 只下载 PDF 和 DOC 文件

# 限速下载
wget --limit-rate=500k -c https://example.com/largefile.iso
# --limit-rate=500k: 限速 500KB/s
# -c: 断点续传

# 使用认证下载
wget --user=admin --password=secret https://secure.example.com/file.zip
# --user=admin: 用户名
# --password=secret: 密码
```

### 1.3 curl vs wget 对比

| 特性 | curl | wget |
|------|------|------|
| 主要用途 | 数据传输、API 测试 | 文件下载 |
| 递归下载 | 不支持 | 支持 |
| 断点续传 | 支持（-C -） | 支持（-c） |
| 协议支持 | 更多（HTTP、FTP、SMTP 等） | HTTP、HTTPS、FTP |
| 输出 | 默认输出到终端 | 默认保存为文件 |
| 后台下载 | 不支持 | 支持（-b） |
| 镜像网站 | 不支持 | 支持（-m） |
| API 交互 | 更强大 | 基础支持 |

## 二、Windows CMD 网络请求

### 2.1 certutil 命令

certutil 是 Windows 内置工具，可用于下载文件。

```cmd
REM 基本语法
certutil -urlcache -split -f [URL] [输出文件]

REM 下载文件
certutil -urlcache -split -f https://example.com/file.zip file.zip
REM certutil: 证书工具命令
REM -urlcache: 使用 URL 缓存
REM -split: 分块下载
REM -f: 强制覆盖（force）
REM https://example.com/file.zip: 源 URL
REM file.zip: 保存的文件名

REM 下载到指定目录
certutil -urlcache -split -f https://example.com/file.zip C:\Downloads\file.zip
REM C:\Downloads\file.zip: 完整路径

REM 示例：下载 Python 安装包
certutil -urlcache -split -f https://www.python.org/ftp/python/3.11.0/python-3.11.0-amd64.exe python.exe
```

### 2.2 bitsadmin 命令

bitsadmin 是 Windows 后台智能传输服务的命令行工具。

```cmd
REM 基本语法
bitsadmin /transfer [任务名] [URL] [输出文件]

REM 下载文件
bitsadmin /transfer myDownload https://example.com/file.zip C:\Downloads\file.zip
REM bitsadmin: 命令名称
REM /transfer: 传输模式
REM myDownload: 任务名称（自定义）
REM https://example.com/file.zip: 源 URL
REM C:\Downloads\file.zip: 目标路径（必须是完整路径）

REM 设置优先级
bitsadmin /transfer myDownload /priority high https://example.com/file.zip C:\file.zip
REM /priority high: 高优先级
REM 优先级：foreground, high, normal, low

REM 示例：下载 Node.js
bitsadmin /transfer nodeDownload https://nodejs.org/dist/v20.0.0/node-v20.0.0-x64.msi C:\Temp\node.msi
```

### 2.3 使用 PowerShell 命令（在 CMD 中）

```cmd
REM 在 CMD 中调用 PowerShell 下载文件
powershell -Command "Invoke-WebRequest -Uri 'https://example.com/file.zip' -OutFile 'file.zip'"
REM powershell: 调用 PowerShell
REM -Command: 执行命令
REM "...": PowerShell 命令（用引号包裹）
REM Invoke-WebRequest: PowerShell 下载命令
REM -Uri: 源 URL
REM -OutFile: 输出文件

REM 显示进度
powershell -Command "$ProgressPreference = 'Continue'; Invoke-WebRequest -Uri 'https://example.com/file.zip' -OutFile 'file.zip'"
REM $ProgressPreference = 'Continue': 显示进度条
```

## 三、Windows PowerShell 网络请求

### 3.1 Invoke-WebRequest 命令详解

Invoke-WebRequest 是 PowerShell 的 HTTP 客户端命令。

#### 基本语法

```powershell
Invoke-WebRequest [-Uri] <String> [参数]
```

#### 基本使用

```powershell
# 获取网页内容
Invoke-WebRequest -Uri "https://example.com"
# Invoke-WebRequest: 命令名称
# -Uri: 目标 URL 参数
# "https://example.com": URL 值

# 简写形式
iwr "https://example.com"
# iwr: Invoke-WebRequest 的别名

# 下载文件
Invoke-WebRequest -Uri "https://example.com/file.zip" -OutFile "file.zip"
# -OutFile "file.zip": 指定输出文件名

# 使用变量
$url = "https://example.com/file.zip"
$output = "C:\Downloads\file.zip"
Invoke-WebRequest -Uri $url -OutFile $output
# $url: 变量存储 URL
# $output: 变量存储输出路径
```

#### 常用参数详解

```powershell
# -Method: 指定 HTTP 方法
Invoke-WebRequest -Uri "https://api.example.com" -Method GET
# -Method GET: 使用 GET 方法
# 可选值：GET, POST, PUT, DELETE, PATCH

# -Headers: 添加请求头
$headers = @{
    "Authorization" = "Bearer token123"
    "Content-Type" = "application/json"
}
Invoke-WebRequest -Uri "https://api.example.com" -Headers $headers
# @{}: 哈希表（键值对集合）
# "Authorization" = "Bearer token123": 头部键值对

# -Body: 发送请求体
$body = @{
    name = "John"
    age = 25
}
Invoke-WebRequest -Uri "https://api.example.com" -Method POST -Body $body
# -Body $body: 发送数据

# -Body 发送 JSON
$json = @{
    name = "John"
    age = 25
} | ConvertTo-Json
Invoke-WebRequest -Uri "https://api.example.com" -Method POST -Body $json -ContentType "application/json"
# ConvertTo-Json: 转换为 JSON 格式
# -ContentType: 指定内容类型

# -UserAgent: 设置 User-Agent
Invoke-WebRequest -Uri "https://example.com" -UserAgent "Mozilla/5.0"
# -UserAgent: 用户代理字符串

# -Proxy: 使用代理
Invoke-WebRequest -Uri "https://example.com" -Proxy "http://proxy:8080"
# -Proxy: 代理服务器地址

# -ProxyCredential: 代理认证
$proxyCred = Get-Credential
Invoke-WebRequest -Uri "https://example.com" -Proxy "http://proxy:8080" -ProxyCredential $proxyCred
# Get-Credential: 获取凭据（弹出对话框）
# -ProxyCredential: 代理认证凭据

# -Credential: HTTP 基本认证
$cred = Get-Credential
Invoke-WebRequest -Uri "https://secure.example.com" -Credential $cred
# -Credential: 认证凭据

# -TimeoutSec: 超时时间
Invoke-WebRequest -Uri "https://example.com" -TimeoutSec 30
# -TimeoutSec 30: 超时 30 秒

# -MaximumRedirection: 最大重定向次数
Invoke-WebRequest -Uri "https://example.com" -MaximumRedirection 5
# -MaximumRedirection 5: 最多重定向 5 次

# -DisableKeepAlive: 禁用保持连接
Invoke-WebRequest -Uri "https://example.com" -DisableKeepAlive
# -DisableKeepAlive: 每次请求后关闭连接

# -UseBasicParsing: 基本解析（不需要 IE）
Invoke-WebRequest -Uri "https://example.com" -UseBasicParsing
# -UseBasicParsing: 不使用 IE 引擎解析
```

#### 实用示例

```powershell
# 下载文件并显示进度
$ProgressPreference = 'Continue'
Invoke-WebRequest -Uri "https://example.com/largefile.zip" -OutFile "file.zip"
# $ProgressPreference = 'Continue': 显示进度条

# 下载 GitHub Release
$url = "https://github.com/user/repo/releases/download/v1.0/app.exe"
$output = "$env:USERPROFILE\Downloads\app.exe"
Invoke-WebRequest -Uri $url -OutFile $output
# $env:USERPROFILE: 用户主目录环境变量

# POST JSON 数据
$headers = @{
    "Content-Type" = "application/json"
    "Authorization" = "Bearer token123"
}
$body = @{
    name = "John"
    email = "john@example.com"
} | ConvertTo-Json

$response = Invoke-WebRequest -Uri "https://api.example.com/users" -Method POST -Headers $headers -Body $body
$response.Content
# $response.Content: 响应内容

# 获取响应头
$response = Invoke-WebRequest -Uri "https://example.com"
$response.Headers
# $response.Headers: 响应头信息

# 获取状态码
$response = Invoke-WebRequest -Uri "https://example.com"
$response.StatusCode
# $response.StatusCode: HTTP 状态码

# 下载并解压
Invoke-WebRequest -Uri "https://example.com/archive.zip" -OutFile "archive.zip"
Expand-Archive -Path "archive.zip" -DestinationPath "extracted"
# Expand-Archive: 解压命令
# -DestinationPath: 解压目标目录

# 批量下载
$urls = @(
    "https://example.com/file1.zip",
    "https://example.com/file2.zip",
    "https://example.com/file3.zip"
)
foreach ($url in $urls) {
    $filename = Split-Path $url -Leaf
    Invoke-WebRequest -Uri $url -OutFile $filename
}
# @(): 数组
# foreach: 循环
# Split-Path -Leaf: 获取文件名
```

### 3.2 Invoke-RestMethod 命令

Invoke-RestMethod 专门用于 REST API 交互，自动解析 JSON/XML。

```powershell
# 基本语法
Invoke-RestMethod [-Uri] <String> [参数]

# GET 请求（自动解析 JSON）
$data = Invoke-RestMethod -Uri "https://api.example.com/users"
$data
# Invoke-RestMethod: 命令名称
# 自动将 JSON 响应转换为 PowerShell 对象

# 简写
irm "https://api.example.com/users"
# irm: Invoke-RestMethod 的别名

# POST 请求
$body = @{
    name = "John"
    age = 25
} | ConvertTo-Json

$result = Invoke-RestMethod -Uri "https://api.example.com/users" -Method POST -Body $body -ContentType "application/json"
# 自动解析响应

# 使用认证
$headers = @{
    "Authorization" = "Bearer token123"
}
$data = Invoke-RestMethod -Uri "https://api.example.com/data" -Headers $headers

# 下载文件（不推荐，使用 Invoke-WebRequest）
Invoke-RestMethod -Uri "https://example.com/file.zip" -OutFile "file.zip"
```

### 3.3 Start-BitsTransfer 命令

Start-BitsTransfer 使用 Windows BITS 服务下载文件，支持断点续传。

```powershell
# 基本语法
Start-BitsTransfer [-Source] <String> [-Destination] <String>

# 下载文件
Start-BitsTransfer -Source "https://example.com/file.zip" -Destination "C:\Downloads\file.zip"
# Start-BitsTransfer: 命令名称
# -Source: 源 URL
# -Destination: 目标路径

# 显示进度
Start-BitsTransfer -Source "https://example.com/largefile.iso" -Destination "C:\file.iso" -DisplayName "Downloading ISO" -Description "Please wait..."
# -DisplayName: 显示名称
# -Description: 描述信息

# 异步下载
Start-BitsTransfer -Source "https://example.com/file.zip" -Destination "C:\file.zip" -Asynchronous
# -Asynchronous: 异步模式（后台下载）

# 设置优先级
Start-BitsTransfer -Source "https://example.com/file.zip" -Destination "C:\file.zip" -Priority High
# -Priority: 优先级（Foreground, High, Normal, Low）

# 断点续传（自动）
Start-BitsTransfer -Source "https://example.com/largefile.iso" -Destination "C:\file.iso"
# BITS 自动支持断点续传
```

## 五、实用场景示例

### 5.1 下载并安装软件

#### Linux

```bash
# 方法 1：使用包管理器（推荐）
sudo apt update
sudo apt install -y nodejs

# 方法 2：下载官方包
wget https://nodejs.org/dist/v20.0.0/node-v20.0.0-linux-x64.tar.xz
tar -xf node-v20.0.0-linux-x64.tar.xz
sudo mv node-v20.0.0-linux-x64 /opt/nodejs
echo 'export PATH="/opt/nodejs/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 方法 3：使用安装脚本
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
# -fsSL: 
#   -f: 失败时不输出（fail silently）
#   -s: 静默模式（silent）
#   -S: 显示错误（show error）
#   -L: 跟随重定向（location）
```

#### Windows PowerShell

```powershell
# 方法 1：使用 winget
winget install --id OpenJS.NodeJS

# 方法 2：下载并安装
$url = "https://nodejs.org/dist/v20.0.0/node-v20.0.0-x64.msi"
$output = "$env:TEMP\nodejs.msi"
Invoke-WebRequest -Uri $url -OutFile $output
Start-Process msiexec.exe -ArgumentList "/i $output /quiet" -Wait
# Start-Process: 启动进程
# msiexec.exe: Windows 安装程序
# /i: 安装
# /quiet: 静默安装
# -Wait: 等待完成

# 方法 3：使用 Chocolatey
choco install -y nodejs
```

### 5.2 下载 GitHub Release

#### Linux

```bash
# 获取最新版本
LATEST=$(curl -s https://api.github.com/repos/user/repo/releases/latest | grep "tag_name" | cut -d '"' -f 4)

# grep "tag_name": 查找标签名
# cut -d '"' -f 4: 提取第 4 个字段

# 下载文件
wget https://github.com/user/repo/releases/download/${LATEST}/app-linux.tar.gz
```

#### Windows PowerShell

```powershell
# 获取最新版本
$repo = "user/repo"

$tag = (Invoke-RestMethod -Uri $releases).tag_name
# Invoke-RestMethod: 自动解析 JSON
# .tag_name: 访问对象属性
```

### 5.3 API 测试

#### Linux

```bash
# GET 请求
curl -X GET https://api.example.com/users

# POST 请求
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"name":"John","email":"john@example.com"}' \
  https://api.example.com/users

  -H "Content-Type: application/json" \
  -d '{"name":"John Updated"}' \

curl -X DELETE https://api.example.com/users/1

# 使用认证
curl -H "Authorization: Bearer token123" https://api.example.com/protected

# 保存响应到文件
curl https://api.example.com/data -o response.json

# 格式化 JSON 输出（需要 jq）
curl https://api.example.com/users | jq '.'
# jq: JSON 处理工具
# '.': 格式化输出
```

#### Windows PowerShell

```powershell
# GET 请求
Invoke-RestMethod -Uri "https://api.example.com/users"

# POST 请求
$body = @{
    name = "John"
    email = "john@example.com"
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://api.example.com/users" -Method POST -Body $body -ContentType "application/json"

# PUT 请求
$body = @{
    name = "John Updated"
} | ConvertTo-Json

Invoke-RestMethod -Uri "https://api.example.com/users/1" -Method PUT -Body $body -ContentType "application/json"

# DELETE 请求
Invoke-RestMethod -Uri "https://api.example.com/users/1" -Method DELETE

# 使用认证
$headers = @{
    "Authorization" = "Bearer token123"
}
Invoke-RestMethod -Uri "https://api.example.com/protected" -Headers $headers

# 保存响应到文件
Invoke-RestMethod -Uri "https://api.example.com/data" | ConvertTo-Json | Out-File response.json
# Out-File: 输出到文件

# 格式化输出
Invoke-RestMethod -Uri "https://api.example.com/users" | ConvertTo-Json -Depth 10
# -Depth 10: JSON 深度
```

### 5.4 批量下载

#### Linux

```bash
# 从文件列表下载
cat urls.txt | while read url; do
  wget "$url"
done
# cat urls.txt: 读取文件
# while read url: 逐行读取
# wget "$url": 下载每个 URL

# 或使用 wget -i
wget -i urls.txt

# 下载序列文件
for i in {1..10}; do
  wget https://example.com/file${i}.zip
done

# 并行下载（使用 xargs）
cat urls.txt | xargs -n 1 -P 4 wget
# xargs: 构建命令
# -n 1: 每次传递 1 个参数
# -P 4: 并行 4 个进程
```

#### Windows PowerShell

```powershell
# 从文件列表下载
Get-Content urls.txt | ForEach-Object {
    $filename = Split-Path $_ -Leaf
    Invoke-WebRequest -Uri $_ -OutFile $filename
}
# Get-Content: 读取文件
# ForEach-Object: 遍历每一行
# $_: 当前项

# 下载序列文件
1..10 | ForEach-Object {
    $url = "https://example.com/file$_.zip"
    Invoke-WebRequest -Uri $url -OutFile "file$_.zip"
}
# 1..10: 生成 1 到 10 的序列

# 并行下载（PowerShell 7+）
$urls = Get-Content urls.txt
$urls | ForEach-Object -Parallel {
    $filename = Split-Path $_ -Leaf
    Invoke-WebRequest -Uri $_ -OutFile $filename
} -ThrottleLimit 4
# -Parallel: 并行执行
# -ThrottleLimit 4: 最多 4 个并行任务
```

## 六、常见问题和解决方案

### 6.1 SSL 证书错误

#### Linux

```bash
# curl 忽略证书验证（不安全）
curl -k https://self-signed.example.com
# -k: 不验证 SSL 证书

# wget 忽略证书验证
wget --no-check-certificate https://self-signed.example.com

# 添加自定义 CA 证书（推荐）
sudo cp custom-ca.crt /usr/local/share/ca-certificates/
sudo update-ca-certificates
```

#### Windows PowerShell

```powershell
# 忽略证书验证（不安全）
[System.Net.ServicePointManager]::ServerCertificateValidationCallback = {$true}
Invoke-WebRequest -Uri "https://self-signed.example.com"

# 或使用 -SkipCertificateCheck（PowerShell 7+）
Invoke-WebRequest -Uri "https://self-signed.example.com" -SkipCertificateCheck
```

### 6.2 下载速度慢

#### Linux

```bash
# 使用镜像源
# 编辑 /etc/apt/sources.list（Ubuntu）
sudo vim /etc/apt/sources.list
# 替换为国内镜像（如阿里云、清华大学）

# 使用代理
export http_proxy=http://proxy:8080
export https_proxy=http://proxy:8080

# 限制速度（避免占用全部带宽）
wget --limit-rate=500k https://example.com/file.zip
curl --limit-rate 500K -O https://example.com/file.zip
```

#### Windows PowerShell

```powershell
# 使用代理
$proxy = "http://proxy:8080"
Invoke-WebRequest -Uri "https://example.com" -Proxy $proxy

# 使用 BITS（支持断点续传，更稳定）
Start-BitsTransfer -Source "https://example.com/file.zip" -Destination "file.zip"
```

### 6.3 下载中断

#### Linux

```bash
# curl 断点续传
curl -C - -O https://example.com/largefile.iso
# -C -: 自动继续下载

# wget 断点续传
wget -c https://example.com/largefile.iso
# -c: 继续下载
```

#### Windows PowerShell

```powershell
# 使用 BITS（自动支持断点续传）
Start-BitsTransfer -Source "https://example.com/largefile.iso" -Destination "file.iso"

# 手动实现断点续传
$url = "https://example.com/largefile.iso"
$output = "file.iso"
$existingSize = (Get-Item $output -ErrorAction SilentlyContinue).Length
if ($existingSize) {
    $headers = @{"Range" = "bytes=$existingSize-"}
    Invoke-WebRequest -Uri $url -OutFile $output -Headers $headers -Resume
}
```

### 6.4 权限问题

#### Linux

```bash
# 使用 sudo 安装包
sudo apt install package-name

# 下载到有权限的目录
wget -P ~/Downloads https://example.com/file.zip

# 修改文件权限
chmod 644 downloaded-file
```

#### Windows PowerShell

```powershell
# 以管理员身份运行 PowerShell
# 右键点击 PowerShell -> 以管理员身份运行

# 或在脚本中检查权限
if (-NOT ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole] "Administrator")) {
    Write-Warning "需要管理员权限"
    exit
}
```
