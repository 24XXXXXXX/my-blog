本文把小程序部署收窄到一条最常见、最适合多平台复用的技术路线：

- 前端：`uni-app + Vue 3`
- 后端：`Spring Boot + Maven`
- 数据库：`MySQL`
- 缓存：`Redis`
- 平台：`微信小程序`、`支付宝小程序`、`抖音小程序`

小程序和 Web 最大的区别是：前端代码不会部署到你的服务器上，而是构建后导入各平台开发者工具，再上传、提审、发布。你的服务器主要负责提供 API、登录、上传、支付回调等能力。

## 二、部署的最终结构

小程序项目最终分成两部分：

### 2.1 服务器上的内容

- `Spring Boot` API
- `MySQL`
- `Redis`
- `Nginx`

### 2.2 平台侧的内容

- 微信小程序代码包
- 支付宝小程序代码包
- 抖音小程序代码包

所以正确顺序永远是：

1. 先把后端部署好
2. 给后端配上 HTTPS 域名
3. 把域名加到小程序后台
4. 再上传小程序前端代码

## 三、个人部署：先部署 Spring Boot 后端

后端这一部分和 Web 全栈项目几乎一致，只是这里我只保留小程序需要的最小步骤。

## 3.1 本地打包 Spring Boot

进入后端项目目录：

```bash
./mvnw clean package -DskipTests
```

打完后会得到：

```bash
target/*.jar
```

## 3.2 服务器安装 Nginx 和 Certbot

登录服务器：

```bash
ssh root@你的服务器IP
```

安装：

```bash
apt update
apt install -y nginx certbot python3-certbot-nginx
```

放行端口：

```bash
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow OpenSSH
ufw enable
```

## 3.3 创建目录并上传 Jar

```bash
mkdir -p /srv/uni-miniapp/backend
mkdir -p /srv/uni-miniapp/logs
```

本地上传：

```bash
scp ./target/*.jar root@你的服务器IP:/srv/uni-miniapp/backend/app.jar
```

## 3.4 创建生产配置

```bash
nano /srv/uni-miniapp/backend/application-prod.yml
```

写入：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/miniapp?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: miniapp
    password: MiniApp@2026#123
  data:
    redis:
      host: 127.0.0.1
      port: 6379
      password: MiniRedis@2026#123

logging:
  file:
    name: /srv/uni-miniapp/logs/app.log
```

这里的数据库和 Redis 配置，请替换成你自己的容器实际配置。

## 3.5 用 systemd 启动 Spring Boot

创建服务文件：

```bash
nano /etc/systemd/system/uni-miniapp.service
```

写入：

```ini
[Unit]
Description=uni-miniapp
After=network.target

[Service]
User=root
WorkingDirectory=/srv/uni-miniapp/backend
ExecStart=/usr/bin/java -jar /srv/uni-miniapp/backend/app.jar --spring.profiles.active=prod --spring.config.additional-location=/srv/uni-miniapp/backend/application-prod.yml
SuccessExitStatus=143
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动：

```bash
systemctl daemon-reload
systemctl enable uni-miniapp
systemctl start uni-miniapp
systemctl status uni-miniapp
```

看日志：

```bash
journalctl -u uni-miniapp -f
```

## 3.6 配置 Nginx 反向代理

创建配置：

```bash
nano /etc/nginx/sites-available/uni-miniapp.conf
```

写入：

```nginx
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用：

```bash
ln -s /etc/nginx/sites-available/uni-miniapp.conf /etc/nginx/sites-enabled/uni-miniapp.conf
nginx -t
systemctl restart nginx
systemctl enable nginx
```

## 3.7 域名解析和 HTTPS

去域名控制台把：

- `api.example.com`

解析到服务器公网 IP。

解析生效后执行：

```bash
certbot --nginx -d api.example.com
certbot renew --dry-run
```

## 3.8 验证后端

```bash
curl https://api.example.com/actuator/health
systemctl status uni-miniapp
nginx -t
```

到这里，后端就部署好了。

## 四、个人部署：构建 uni-app 小程序包

这一部分在你本地电脑完成。

## 4.1 配置生产环境变量

在 uni-app 项目里准备生产环境，例如：

```env
VITE_API_BASE_URL=https://api.example.com
```

如果你是写在常量文件里，也要确保正式包用的是生产地址。

## 4.2 安装依赖

```bash
npm install
```

## 4.3 构建微信小程序

```bash
npm run build:mp-weixin
```

默认情况下，uni-app Vite 项目常见输出目录是：

```bash
dist/build/mp-weixin
```

## 4.4 构建支付宝小程序

```bash
npm run build:mp-alipay
```

输出目录通常是：

```bash
dist/build/mp-alipay
```

## 4.5 构建抖音小程序

```bash
npm run build:mp-toutiao
```

输出目录通常是：

```bash
dist/build/mp-toutiao
```

如果你的脚本名不同，以你项目 `package.json` 里的 `scripts` 为准。

## 五、把域名配置到平台后台

这一部分很关键。没配合法域名，真机几乎一定调不通。

## 5.1 微信小程序

进入微信小程序后台，配置：

- `request 合法域名`：`https://api.example.com`
- 如果有上传文件：配置上传域名
- 如果有下载文件：配置下载域名
- 如果有 `WebSocket`：配置 `wss://` 域名

## 5.2 支付宝小程序

进入支付宝开放平台后台，配置：

- 请求域名
- 上传域名
- 下载域名

## 5.3 抖音小程序

进入抖音开放平台后台，配置：

- 请求域名
- 上传域名
- 下载域名
- WebSocket 域名

注意两件事：

1. 正式环境一般必须是 `HTTPS`
2. 配的是域名，不是 IP

## 六、上传代码和发布

## 6.1 微信小程序上传步骤

1. 打开微信开发者工具
2. 导入目录 `dist/build/mp-weixin`
3. 选择正确的 `AppID`
4. 在真机调试里验证登录、接口、上传、支付
5. 点击“上传”
6. 回到微信小程序后台提交审核
7. 审核通过后发布

## 6.2 支付宝小程序上传步骤

1. 打开支付宝开发者工具
2. 导入目录 `dist/build/mp-alipay`
3. 绑定正确应用
4. 真机或模拟器验证
5. 上传版本
6. 在后台提交审核
7. 审核通过后发布

## 6.3 抖音小程序上传步骤

1. 打开抖音开发者工具
2. 导入目录 `dist/build/mp-toutiao`
3. 选择正确应用
4. 真机调试验证
5. 上传版本
6. 后台提交审核
7. 审核通过后发布

## 七、多人/团队部署

团队阶段仍然推荐同一套技术栈：

- 前端：`uni-app + Vue 3`
- 后端：`Spring Boot`
- 数据：`MySQL + Redis`

但发布方式要改成：

1. 后端容器化
2. 测试环境和生产环境分开
3. 小程序前端分别出测试包和正式包

## 7.1 团队推荐目录

```bash
/srv/uni-miniapp/
  backend/
  docker-compose.yml
```

## 7.2 后端 Dockerfile

```dockerfile
FROM maven:3.9.9-eclipse-temurin-21 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-jar","/app/app.jar","--spring.profiles.active=prod"]
```

## 7.3 团队 `docker-compose.yml`

```yaml
version: "3.9"

services:
  api:
    build: ./backend
    container_name: uni-miniapp-api
    restart: unless-stopped
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://host.docker.internal:3306/miniapp?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_USERNAME: miniapp
      SPRING_DATASOURCE_PASSWORD: MiniApp@2026#123
      SPRING_DATA_REDIS_HOST: host.docker.internal
      SPRING_DATA_REDIS_PORT: 6379
      SPRING_DATA_REDIS_PASSWORD: MiniRedis@2026#123
    extra_hosts:
      - "host.docker.internal:host-gateway"
    ports:
      - "8080:8080"
```

启动：

```bash
docker compose up -d --build
docker compose logs -f api
```

## 7.4 团队阶段的关键原则

后端必须先上兼容版本，再提审前端。

原因很简单：

- 小程序审核时间不稳定
- 线上永远可能同时存在旧版用户和新版用户

所以后端接口不要做破坏性修改。

## 八、标准商业化部署

商业化阶段推荐：

- 前端：`uni-app` 多平台构建
- 后端：`Kubernetes`
- 数据：托管 MySQL、托管 Redis
- 文件：对象存储
- 发布：灰度开关 + 回滚

## 8.1 最小 K8s 后端部署

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: uni-miniapp-api
spec:
  replicas: 2
  selector:
    matchLabels:
      app: uni-miniapp-api
  template:
    metadata:
      labels:
        app: uni-miniapp-api
    spec:
      containers:
        - name: uni-miniapp-api
          image: registry.example.com/uni-miniapp-api:1.0.0
          ports:
            - containerPort: 8080
```

部署：

```bash
kubectl apply -f deployment.yaml
```

## 九、最容易踩的坑

1. 后端没上 HTTPS 就开始真机调试
2. 配了请求域名，没配上传下载域名
3. 小程序前端已经切生产环境，后端还没发布
4. 线上包还连着测试环境 API
5. 提审前只测了模拟器，没测真机

## 总结

这套 `uni-app + Vue 3 + Spring Boot + MySQL + Redis` 的小程序部署，最稳顺序是：

1. 先按相关文章准备好 Docker、MySQL、Redis、JDK、Node
2. 先把 Spring Boot 后端部署到服务器
3. 给 API 域名配上 HTTPS
4. 去各平台后台配置合法域名
5. 本地构建 uni-app 的微信、支付宝、抖音包
6. 导入开发者工具，上传、提审、发布

小程序部署最关键的一点，是永远先把后端和域名准备好，再做平台上传。顺序反了，排错会非常痛苦。
