## 一、命令行参数基础

### 1.1 什么是命令行参数

命令行参数（Command-line Arguments）是在执行命令时传递给程序的选项和参数，用于控制程序的行为。

### 1.2 参数的类型

#### 短选项（Short Options）
- 使用单个连字符 `-` 开头
- 通常是单个字母
- 可以组合使用
- 示例：`-a`, `-l`, `-R`, `-rf`

#### 长选项（Long Options）
- 使用双连字符 `--` 开头
- 通常是完整单词
- 更具可读性
- 示例：`--all`, `--recursive`, `--help`

#### 参数值
- 某些选项需要提供值
- 示例：`-p 8080`, `--port=8080`

### 1.3 参数的表示方法

```bash
# 短选项
command -a -b -c

# 短选项组合
command -abc

# 短选项带值
command -p 8080
command -p8080

# 长选项
command --all --verbose

# 长选项带值
command --port 8080
command --port=8080

# 混合使用
command -a --verbose -p 8080
```

## 二、通用命令行参数

### 2.1 帮助和版本信息

#### `-h` / `--help`
显示命令的帮助信息

```bash
# Linux
ls --help
grep -h
docker --help

# Windows PowerShell
Get-Help Get-Process
Get-Command -?
```

#### `-v` / `--version`
显示命令的版本信息

```bash
# Linux
python --version
python -V
docker --version

# Windows
python --version
node --version
```

#### `--verbose`
显示详细输出信息

```bash
# Linux
rsync --verbose source/ dest/
apt-get install --verbose package

# Windows PowerShell
Copy-Item -Verbose source dest
```

## 三、文件和目录操作参数

### 3.1 递归操作

#### `-r` / `-R` / `--recursive`
递归处理目录及其子目录

```bash
# Linux - 复制目录
cp -r source/ dest/
cp -R source/ dest/

# Linux - 删除目录
rm -r directory/
rm -R directory/

# Linux - 修改权限
chmod -R 755 directory/
chown -R user:group directory/

# Linux - 搜索
grep -r "pattern" directory/

# Windows PowerShell - 递归复制
Copy-Item -Recurse source dest

# Windows PowerShell - 递归删除
Remove-Item -Recurse directory

# Windows CMD
xcopy /E /I source dest
```

**使用场景：**
- 复制整个目录树
- 删除目录及其所有内容
- 递归修改目录权限
- 在目录树中搜索文件

### 3.2 强制操作

#### `-f` / `--force`
强制执行操作，不提示确认

```bash
# Linux - 强制删除
rm -f file.txt
rm -rf directory/

# Linux - 强制复制（覆盖）
cp -f source dest

# Linux - 强制移动
mv -f source dest

# Windows PowerShell - 强制操作
Remove-Item -Force file.txt
Copy-Item -Force source dest

# Windows CMD
del /F file.txt
```

**注意：** 使用 `-f` 参数要特别小心，因为它会跳过确认提示，可能导致数据丢失。

### 3.3 交互式操作

#### `-i` / `--interactive`
在执行操作前提示确认

```bash
# Linux - 交互式删除
rm -i file.txt
rm -ri directory/

# Linux - 交互式复制
cp -i source dest

# Linux - 交互式移动
mv -i source dest

# Windows PowerShell
Remove-Item -Confirm file.txt
```

**使用场景：**
- 删除重要文件前确认
- 避免意外覆盖文件
- 批量操作时逐个确认

### 3.4 详细输出

#### `-v` / `--verbose`
显示详细的操作信息

```bash
# Linux - 详细复制
cp -v source dest

# Linux - 详细删除
rm -v file.txt

# Linux - 详细移动
mv -v source dest

# Linux - 详细权限修改
chmod -v 755 file.txt
chown -v user:group file.txt

# Windows PowerShell
Copy-Item -Verbose source dest
Remove-Item -Verbose file.txt
```

### 3.5 保留属性

#### `-p` / `--preserve`
保留文件属性（权限、时间戳等）

```bash
# Linux - 保留属性复制
cp -p source dest

# Linux - 保留所有属性
cp -a source dest  # 等同于 -dR --preserve=all

# rsync 保留属性
rsync -a source/ dest/

# Windows PowerShell
Copy-Item -Path source -Destination dest -Force
```

**保留的属性包括：**
- 文件权限（mode）
- 所有者和组（ownership）
- 时间戳（timestamps）
- 扩展属性（extended attributes）
- ACL（访问控制列表）

### 3.6 创建父目录

#### `-p` / `--parents`
创建目录时自动创建父目录

```bash
# Linux - 递归创建目录
mkdir -p /path/to/deep/directory

# 如果父目录不存在，会自动创建
mkdir -p project/src/main/java

# Windows PowerShell
New-Item -ItemType Directory -Force -Path "C:\path\to\directory"

# Windows CMD
mkdir "C:\path\to\directory"  # 自动创建父目录
```

## 四、文件查看和搜索参数

### 4.1 行号显示

#### `-n` / `--line-number`
显示行号

```bash
# Linux - cat 显示行号
cat -n file.txt

# Linux - grep 显示行号
grep -n "pattern" file.txt

# Linux - less 显示行号
less -N file.txt

# Windows PowerShell
Get-Content file.txt | ForEach-Object {"{0}: {1}" -f $_.ReadCount, $_}
```

### 4.2 忽略大小写

#### `-i` / `--ignore-case`
搜索时忽略大小写

```bash
# Linux - grep 忽略大小写
grep -i "pattern" file.txt

# Linux - less 忽略大小写
less -i file.txt

# Linux - sort 忽略大小写
sort -i file.txt

# Windows PowerShell
Select-String -Pattern "pattern" -Path file.txt -CaseSensitive:$false
```

### 4.3 反向匹配

#### `-v` / `--invert-match`
显示不匹配的行

```bash
# Linux - grep 反向匹配
grep -v "pattern" file.txt

# 过滤注释行
grep -v "^#" config.conf

# 过滤空行
grep -v "^$" file.txt

# 组合使用：过滤注释和空行
grep -v "^#" config.conf | grep -v "^$"
```

### 4.4 统计

#### `-c` / `--count`
统计匹配的行数或字符数

```bash
# Linux - grep 统计匹配行数
grep -c "pattern" file.txt

# Linux - wc 统计
wc -l file.txt    # 行数
wc -w file.txt    # 单词数
wc -c file.txt    # 字节数
wc -m file.txt    # 字符数

# Windows PowerShell
(Get-Content file.txt | Select-String "pattern").Count
```

### 4.5 上下文显示

#### `-A` / `--after-context`
显示匹配行及其后 N 行

```bash
# Linux - 显示匹配行及后 3 行
grep -A 3 "pattern" file.txt
```

#### `-B` / `--before-context`
显示匹配行及其前 N 行

```bash
# Linux - 显示匹配行及前 3 行
grep -B 3 "pattern" file.txt
```

#### `-C` / `--context`
显示匹配行及其前后 N 行

```bash
# Linux - 显示匹配行及前后 3 行
grep -C 3 "pattern" file.txt
```

## 五、网络和端口参数

### 5.1 端口指定

#### `-p` / `--port`
指定端口号

```bash
# Linux - SSH 指定端口
ssh -p 2222 user@host

# Linux - MySQL 指定端口
mysql -h localhost -P 3306 -u root -p

# Docker 端口映射
docker run -p 8080:80 nginx
docker run -p 3306:3306 mysql

# Windows PowerShell - 测试端口
Test-NetConnection -ComputerName host -Port 80
```

### 5.2 主机指定

#### `-h` / `--host`
指定主机名或 IP 地址

```bash
# Linux - MySQL 连接
mysql -h 192.168.1.100 -u root -p

# Linux - Redis 连接
redis-cli -h localhost -p 6379

# Windows PowerShell
Test-Connection -ComputerName hostname
```

### 5.3 用户指定

#### `-u` / `--user`
指定用户名

```bash
# Linux - MySQL 用户
mysql -u root -p

# Linux - 以指定用户运行
sudo -u username command

# Docker 指定用户
docker run -u 1000:1000 image

# Windows PowerShell
Start-Process -Credential username command
```

## 六、压缩和归档参数

### 6.1 tar 命令参数

#### `-c` / `--create`
创建归档文件

```bash
tar -c -f archive.tar files/
tar -cf archive.tar files/
```

#### `-x` / `--extract`
解压归档文件

```bash
tar -x -f archive.tar
tar -xf archive.tar
```

#### `-v` / `--verbose`
显示详细信息

```bash
tar -cvf archive.tar files/
tar -xvf archive.tar
```

#### `-f` / `--file`
指定归档文件名

```bash
tar -cf archive.tar files/
```

#### `-z` / `--gzip`
使用 gzip 压缩

```bash
tar -czf archive.tar.gz files/
tar -xzf archive.tar.gz
```

#### `-j` / `--bzip2`
使用 bzip2 压缩

```bash
tar -cjf archive.tar.bz2 files/
tar -xjf archive.tar.bz2
```

#### `-J` / `--xz`
使用 xz 压缩

```bash
tar -cJf archive.tar.xz files/
tar -xJf archive.tar.xz
```

#### `-C` / `--directory`
指定解压目录

```bash
tar -xf archive.tar -C /path/to/directory
```

### 6.2 压缩命令参数

```bash
# gzip
gzip -c file > file.gz      # 保留原文件
gzip -d file.gz              # 解压
gzip -k file                 # 保留原文件（-k, --keep）
gzip -9 file                 # 最大压缩率

# zip
zip -r archive.zip directory/    # 递归压缩
zip -e archive.zip files         # 加密压缩
unzip archive.zip                # 解压
unzip -d /path archive.zip       # 解压到指定目录

# Windows PowerShell
Compress-Archive -Path files -DestinationPath archive.zip
Expand-Archive -Path archive.zip -DestinationPath dest
```

## 七、进程和系统参数

### 7.1 后台运行

#### `-d` / `--detach`
后台运行（守护进程模式）

```bash
# Docker 后台运行
docker run -d nginx

# Linux - 后台运行命令
command &
nohup command &

# Windows PowerShell - 后台作业
Start-Job -ScriptBlock { command }
```

### 7.2 信号发送

#### `-9` / `-KILL`
强制终止进程

```bash
# Linux - 发送信号
kill -9 PID
kill -KILL PID
kill -SIGKILL PID

# 其他常用信号
kill -15 PID    # SIGTERM（默认，优雅终止）
kill -1 PID     # SIGHUP（重新加载配置）
kill -2 PID     # SIGINT（Ctrl+C）

# Windows PowerShell
Stop-Process -Id PID -Force
```

### 7.3 进程查看

#### `-a` / `--all`
显示所有进程

```bash
# Linux
ps -a           # 所有终端的进程
ps aux          # 所有用户的所有进程
docker ps -a    # 所有容器（包括停止的）

# Windows PowerShell
Get-Process
```

#### `-e` / `--everyone`
显示所有进程

```bash
# Linux
ps -e
ps -ef
```

### 8.1 权限模式

#### `-m` / `--mode`
设置权限模式

```bash
# Linux - 创建目录并设置权限
mkdir -m 755 directory
```

## 九、Docker 常用参数

### 9.1 容器运行参数

```bash
# -d, --detach: 后台运行
docker run -d nginx

# -it: 交互式终端
docker run -it ubuntu /bin/bash

# --name: 指定容器名称
docker run --name my-nginx nginx

# -p, --publish: 端口映射

# -v, --volume: 挂载卷
docker run -v /host/path:/container/path nginx

# -e, --env: 设置环境变量
docker run -e MYSQL_ROOT_PASSWORD=password mysql

# --rm: 容器退出后自动删除
docker run --rm nginx

# --restart: 重启策略
docker run --restart unless-stopped nginx

# --memory: 限制内存
docker run --memory 1g nginx

# --cpus: 限制 CPU
docker run --cpus 2 nginx

# --network: 指定网络
docker run --network my-network nginx
```

## 十、Git 常用参数

### 10.1 提交参数

```bash
# -m, --message: 提交信息
git commit -m "commit message"

# -a, --all: 提交所有修改
git commit -a -m "message"
```

```bash
# -r, --remote: 远程分支
git branch -r

# -a, --all: 所有分支
git branch -a
```

### 10.3 推送和拉取参数


## 十三、最佳实践

### 13.1 参数使用建议

1. **优先使用长选项**
   ```bash
   # 更易读
   grep --recursive --ignore-case "pattern" directory/
   
   # 而不是
   grep -ri "pattern" directory/
   ```

2. **危险操作使用交互式**
   ```bash
   # 删除前确认
   rm -i important_file.txt
   rm -ri important_directory/
   ```

3. **批量操作使用详细输出**
   ```bash
   # 查看操作过程
   cp -rv source/ dest/
   chmod -Rv 755 directory/
   ```

4. **组合使用短选项**
   ```bash
   # 可以组合
   tar -czvf archive.tar.gz files/
   
   # 等同于
   tar -c -z -v -f archive.tar.gz files/
   ```

### 13.2 安全建议

1. **避免盲目使用 `-f`**
   ```bash
   # 危险：可能删除重要文件
   rm -rf /
   
   # 安全：先确认
   rm -ri directory/
   ```

2. **使用 `--` 分隔选项和参数**
   ```bash
   # 处理以 - 开头的文件名
   rm -- -filename
   ```

3. **引用包含空格的参数**
   ```bash
   # 正确
   mkdir "My Documents"
   
   # 错误
   mkdir My Documents  # 会创建两个目录
   ```

## 十四、总结

本文详细介绍了 Windows 和 Linux 系统中常用的命令行参数，包括：

1. **通用参数**：帮助、版本、详细输出
2. **文件操作参数**：递归、强制、交互、保留属性
3. **搜索参数**：行号、大小写、反向匹配、上下文
4. **网络参数**：端口、主机、用户
5. **压缩归档参数**：创建、解压、压缩格式
6. **进程参数**：后台运行、信号、查看
7. **Docker 参数**：容器运行、端口映射、卷挂载
8. **Git 参数**：提交、分支、推送

### 关键要点

- 短选项使用单个 `-`，长选项使用双 `--`
- 短选项可以组合使用
- 危险操作使用 `-i` 交互式确认
- 递归操作使用 `-r` 或 `-R`
- 强制操作使用 `-f` 要谨慎
- 详细输出使用 `-v` 便于调试
- Windows PowerShell 参数通常更具描述性
