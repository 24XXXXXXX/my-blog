本文把桌面应用部署收窄到一条主流组合：

- 客户端：`Electron + Vue 3`
- 后端：`Spring Boot + Maven`
- 数据库：`MySQL`
- 缓存：`Redis`
- 下载与更新：`Nginx`

这类项目的部署，不只是把后端上线，还包括：

1. 打包安装包
2. 把安装包和更新文件发布到服务器
3. 部署后端 API
4. 让客户端能检查更新

基础环境如果还没准备好，请先看这些文章：

- [Docker 安装和命令大全](./docker安装和命令.md)
- [在 Docker 中部署 MySQL](./在Docker中部署MySQL.md)

- [在 Windows 和 Ubuntu 下使用 NVM 管理 Node.js 多版本](./在Windows和Ubuntu下使用NVM管理Node.js多版本.md)

本文默认：

1. 服务器系统是 `Ubuntu 22.04`
2. `MySQL` 和 `Redis` 已经用 Docker 跑起来
3. 服务器已经有 `JDK 21`
4. 本地已经有 `Node.js`
5. 你的 Electron 项目本地已经能正常运行

统一域名：

- API：`api.example.com`
- 下载和更新：`download.example.com`

## 二、最终部署结构

推荐的目录结构：

```bash
/srv/electron-vue/
  backend/
    app.jar
  downloads/
    stable/
    beta/
  logs/
```

这里的含义是：

- `stable/` 放正式版安装包和更新文件
- `beta/` 放测试版安装包和更新文件

## 三、个人部署：先部署后端 API

如果你的桌面应用有登录、同步、会员、AI、上传下载，这一部分就是必须的。

## 3.1 本地打包 Spring Boot

```bash
./mvnw clean package -DskipTests
```

生成：

```bash
target/*.jar
```

## 3.2 服务器安装 Nginx 和 Certbot

```bash
ssh root@你的服务器IP
apt update
apt install -y nginx certbot python3-certbot-nginx
```

放行端口：

```bash
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow OpenSSH
ufw enable
```

## 3.3 创建目录并上传 Jar

```bash
mkdir -p /srv/electron-vue/backend
mkdir -p /srv/electron-vue/downloads/stable
mkdir -p /srv/electron-vue/downloads/beta
mkdir -p /srv/electron-vue/logs
```

本地上传：

```bash
scp ./target/*.jar root@你的服务器IP:/srv/electron-vue/backend/app.jar
```

## 3.4 创建生产配置

```bash
nano /srv/electron-vue/backend/application-prod.yml
```

写入：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/electron_app?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: electron_app
    password: ElectronApp@2026#123
  data:
    redis:
      host: 127.0.0.1
      port: 6379
      password: ElectronRedis@2026#123

logging:
  file:
    name: /srv/electron-vue/logs/app.log
```

## 3.5 用 systemd 启动 Spring Boot

```bash
nano /etc/systemd/system/electron-vue.service
```

写入：

```ini
[Unit]
Description=electron-vue
After=network.target

[Service]
User=root
WorkingDirectory=/srv/electron-vue/backend
ExecStart=/usr/bin/java -jar /srv/electron-vue/backend/app.jar --spring.profiles.active=prod --spring.config.additional-location=/srv/electron-vue/backend/application-prod.yml
SuccessExitStatus=143
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动：

```bash
systemctl daemon-reload
systemctl enable electron-vue
systemctl start electron-vue
systemctl status electron-vue
```

## 3.6 配置 Nginx

```bash
nano /etc/nginx/sites-available/electron-vue.conf
```

写入：

```nginx
server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name download.example.com;

    root /srv/electron-vue/downloads;
    index index.html;

    location / {
        autoindex on;
    }
}
```

启用：

```bash
ln -s /etc/nginx/sites-available/electron-vue.conf /etc/nginx/sites-enabled/electron-vue.conf
nginx -t
systemctl restart nginx
systemctl enable nginx
```

## 3.7 配置 HTTPS

把：

- `api.example.com`
- `download.example.com`

解析到服务器公网 IP，然后执行：

```bash
certbot --nginx -d api.example.com -d download.example.com
certbot renew --dry-run
```

## 3.8 验证后端

```bash
curl https://api.example.com/actuator/health
systemctl status electron-vue
```

## 四、个人部署：构建 Electron 安装包

这部分在本地电脑执行。

## 4.1 安装依赖

```bash
npm install
```

如果你还没装自动更新相关依赖：

```bash
npm install -D electron-builder
npm install electron-updater
```

## 4.2 配置 Electron Builder

在 `package.json` 中加入一个最小可用配置：

```json
{
  "build": {
    "appId": "com.example.electronvue",
    "productName": "ElectronVueApp",
    "directories": {
      "output": "release"
    },
    "publish": [
      {
        "provider": "generic",
        "url": "https://download.example.com/stable"
      }
    ],
    "win": {
      "target": ["nsis"]
    }
  }
}
```

## 4.3 在客户端接入更新逻辑

主进程示例：

```javascript
const { app, BrowserWindow, dialog } = require("electron");
const { autoUpdater } = require("electron-updater");

function createWindow() {
  const win = new BrowserWindow({
    width: 1280,
    height: 800
  });

  win.loadURL("http://localhost:5173");
}

app.whenReady().then(() => {
  createWindow();
  autoUpdater.checkForUpdatesAndNotify();
});

autoUpdater.on("update-downloaded", async () => {
  await dialog.showMessageBox({
    type: "info",
    message: "更新已下载完成，点击确定后重启安装"
  });
  autoUpdater.quitAndInstall();
});
```

生产环境里如果你不是直接加载开发服务器，请把 `loadURL` 改成你自己的正式加载方式。

## 4.4 构建 Windows 安装包

```bash
npx electron-builder --win nsis
```

常见输出在：

```bash
release/
```

里面通常会有：

- `ElectronVueApp Setup 1.0.0.exe`
- `latest.yml`

## 4.5 上传正式版安装包和更新文件

```bash
scp release/* root@你的服务器IP:/srv/electron-vue/downloads/stable/
```

上传后测试：

```text
https://download.example.com/stable/latest.yml
https://download.example.com/stable/ElectronVueApp%20Setup%201.0.0.exe
```

## 4.6 测试版发布

如果你要给测试人员先用，把 `publish.url` 改成：

```text
https://download.example.com/beta
```

然后上传到：

```bash
/srv/electron-vue/downloads/beta/
```

这样测试版和正式版就分开了。

## 五、验证自动更新

自动更新的最小验证方式：

1. 先安装 `1.0.0`
2. 把项目版本改成 `1.0.1`
3. 重新执行打包命令
4. 再把新的安装包和 `latest.yml` 上传到 `stable/`
5. 打开老版本客户端，看是否提示升级

第二次发布命令和第一次一样：

```bash
npx electron-builder --win nsis
scp release/* root@你的服务器IP:/srv/electron-vue/downloads/stable/
```

## 六、多人/团队部署

团队阶段推荐仍然用：

- `Electron + Vue 3`
- `Spring Boot`
- `MySQL + Redis`

只是把发布方式改成：

1. 后端容器化
2. 客户端自动打包
3. 正式版和测试版分目录

## 6.1 后端 Dockerfile

```dockerfile
FROM maven:3.9.9-eclipse-temurin-21 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

FROM eclipse-temurin:21-jre
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-jar","/app/app.jar","--spring.profiles.active=prod"]
```

## 6.2 `docker-compose.yml`

```yaml
version: "3.9"

services:
  api:
    build: ./backend
    container_name: electron-vue-api
    restart: unless-stopped
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://host.docker.internal:3306/electron_app?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_USERNAME: electron_app
      SPRING_DATASOURCE_PASSWORD: ElectronApp@2026#123
      SPRING_DATA_REDIS_HOST: host.docker.internal
      SPRING_DATA_REDIS_PORT: 6379
      SPRING_DATA_REDIS_PASSWORD: ElectronRedis@2026#123
    extra_hosts:
      - "host.docker.internal:host-gateway"
    ports:
      - "8080:8080"
```

启动：

```bash
docker compose up -d --build
docker compose logs -f api
```

## 6.3 客户端自动打包示例

```yaml
name: build-desktop

on:
  push:
    branches:
      - main

jobs:
  build-win:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npx electron-builder --win nsis
```

## 七、标准商业化部署

商业化阶段推荐：

- 后端：`Kubernetes`
- 数据：托管 MySQL、托管 Redis
- 下载：对象存储 + CDN
- 更新：正式版、灰度版、回滚版分渠道

至少要做到：

1. 安装包签名
2. 更新文件可回滚
3. 客户端崩溃可监控
4. 下载文件走 HTTPS

---

## 八、最容易踩的坑

1. 上传了安装包，但忘了上传 `latest.yml`
2. `latest.yml` 版本号没变
3. 测试版和正式版混在一个目录
4. Electron 前端还写着本地开发地址
5. 后端接口没上 HTTPS

## 总结

这套 `Electron + Vue 3 + Spring Boot + MySQL + Redis` 的桌面应用部署，最稳顺序是：

1. 先按相关文章准备好 Docker、MySQL、Redis、JDK、Node
2. 先部署 Spring Boot 后端
3. 再配置下载和更新域名
4. 本地打包 Electron 安装包
5. 上传安装包和 `latest.yml`
6. 先手工下载安装，再测试自动更新

桌面应用最怕的是一上来就做自动更新，结果连基础下载链路都没跑通。先把“能下载、能安装、能联网”做稳，再补自动更新和灰度发布，成功率最高。
