# 使用 Git 推送到 GitHub

## 0.准备工作

1. 在 GitHub 上新建一个仓库（拿到仓库地址）。
2. 本地已安装 Git。

## 1.初始化仓库（如果还未初始化）

```bash
git init
```

## 2.配置用户名和邮箱（首次使用建议先配置）

全局配置（推荐）：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

只对当前仓库生效（可选）：

```bash
git config user.name "你的名字"
git config user.email "你的邮箱"
```

查看是否配置成功：

```bash
git config --global --list
```

## 3.关联远程仓库（需在GitHub上新建仓库）

HTTPS：

```bash
git remote add origin https://github.com/<user>/<repo>.git
```

SSH：

```bash
git remote add origin git@github.com:<user>/<repo>.git
```

检查一下是否绑定成功：

```bash
git remote -v
```

## 4.添加文件到暂存区（git add）

### 常用用法

添加单个文件：

```bash
git add <文件名>
```

例如：

```bash
git add index.html
```

添加当前目录下所有变更（不包含删除）：

```bash
git add .
```

添加所有变更（包含删除）：

```bash
git add -A
```

等同于：

```bash
git add --all
```

仅添加已被 Git 跟踪的文件（修改或删除，不包含新文件）：

```bash
git add -u
```

等同于：

```bash
git add --update
```

交互式添加（只提交部分改动）：

```bash
git add -p
```

等同于：

```bash
git add --patch
```

### 小技巧

提交前通常先看一下：

```bash
git status
```

`git add` 不会添加被 `.gitignore` 忽略的文件。

如果需要更详细的说明：

```bash
git add --help
```

## 5.提交（git commit）

```bash
git commit -m "feat: init"
```

## 6.推送到 GitHub（git push）

如果你希望主分支叫 `main`：

```bash
git branch -M main
```

首次推送并设置上游分支：

```bash
git push -u origin main
```
