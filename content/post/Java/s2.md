对比 `mvn spring-boot:run` 与 `mvn -DskipTests clean spring-boot:run` 在清理构建产物、测试执行与启动体验上的区别。
<!--more-->

## 核心区别总结

最核心的区别在于 `-DskipTests` 参数和 `clean` 生命周期阶段。

- `-DskipTests`：跳过执行测试（通常仍会编译测试代码）。
- `clean`：清理之前构建生成的文件（例如 `target/`）。

## 命令详解

### 1) `mvn spring-boot:run`

这是一个最直接的启动命令。

工作流程（概念上）：

- 执行 `spring-boot-maven-plugin` 的 `run` 目标。
- 确保项目代码已编译（至少主代码会被编译/更新），然后启动内嵌服务器运行应用。

关键点：

- **一般不会执行单元测试**：单元测试属于 `test` 阶段，`spring-boot:run` 通常不会跑到 `test` 阶段，因此不会因为测试失败而无法启动。
- **但可能会编译测试代码**：很多场景下会触发到 `test-compile` 或解析 test scope 依赖（取决于插件/项目配置），因此测试源码语法错误仍可能导致构建失败。
- **不做清理**：不会删除 `target/`，属于增量编译/增量构建模式。

### 2) `mvn -DskipTests clean spring-boot:run`

这是一个更“干净”、更稳的启动命令，开发阶段很常用。

工作流程：

- `clean`：先删除 `target/`，确保从零开始。
- `-DskipTests`：跳过测试执行。
- `spring-boot:run`：在全新的编译输出上启动应用。

为什么这样组合更常用：

- **更可控**：`clean` 能避免旧产物（旧 class / 旧资源文件 / 旧生成代码）造成的奇怪问题。
- **更快**：如果你的构建链里存在会执行测试的环节，`-DskipTests` 能显著减少启动等待。

## 对比表格

| 特性 | `mvn spring-boot:run` | `mvn -DskipTests clean spring-boot:run` |
| --- | --- | --- |
| 清理旧构建 | 否（保留 `target/`） | 是（删除 `target/`） |
| 编译主代码 | 是（增量/按需） | 是（全量） |
| 执行测试 | 通常否 | 否（显式跳过） |
| 启动速度 | 通常更快，但可能受旧产物影响 | 通常更稳，首次可能略慢 |
| 适用场景 | 日常快速启动 | 遇到构建缓存/资源不刷新/生成代码等问题时更推荐 |

## 其他常见变体

### `mvn clean spring-boot:run`

只清理，不额外声明跳过测试。通常也不会跑测试，但如果你的构建链中绑定了测试相关插件/阶段，它会按配置执行。

### `mvn -Dmaven.test.skip=true clean spring-boot:run`

`-Dmaven.test.skip=true` 会更“狠”一些：**测试编译和测试执行都跳过**（比 `-DskipTests` 更彻底）。

## 结论与建议

开发阶段更推荐：

```bash
mvn -DskipTests clean spring-boot:run
```

提交代码前（确保测试都通过）再单独跑：

```bash
mvn clean test
```
