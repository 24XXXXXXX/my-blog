`Map<Integer, Integer>` 中的两个 `Integer` 分别是 Key 和 Value，它们的业务含义由上下文决定；同时补充 `HashMap` 的底层原理与常见坑。

本文主要讲清楚两件事：

- `HashMap` / `Map` 的本质是什么
- `Map<Integer, Integer>` 里的两个 `Integer` 到底分别代表什么

## HashMap<>() 的本质

`HashMap<K, V>` 是一个键值对（Key-Value）存储结构，存储的是“映射关系”，而不是“索引和值”。

推荐写法（面向接口编程）：

```java
import java.util.HashMap;
import java.util.Map;

Map<String, Integer> map = new HashMap<>();
```

## Map<Integer, Integer> 的实际含义

对于 `Map<Integer, Integer>`：

- 第一个 `Integer` 是 **键（Key）**
- 第二个 `Integer` 是 **值（Value）**

它们的业务含义完全取决于上下文。

### 示例1：数组值 -> 出现次数（频率统计）

```java
import java.util.HashMap;
import java.util.Map;

int[] nums = {1, 2, 2, 3, 3, 3};
Map<Integer, Integer> frequencyMap = new HashMap<>();

for (int num : nums) {
    frequencyMap.put(num, frequencyMap.getOrDefault(num, 0) + 1);
}
// 结果：{1=1, 2=2, 3=3}
// 键：数组中的数字；值：该数字出现的次数
```

### 示例2：元素值 -> 数组索引

```java
import java.util.HashMap;
import java.util.Map;

int[] nums = {10, 20, 30, 40};
Map<Integer, Integer> indexMap = new HashMap<>();

for (int i = 0; i < nums.length; i++) {
    indexMap.put(nums[i], i);
}
// 结果：{10=0, 20=1, 30=2, 40=3}
// 键：数组元素值；值：该值在数组中的索引
```

### 示例3：用户ID -> 年龄

```java
import java.util.HashMap;
import java.util.Map;

Map<Integer, Integer> userAges = new HashMap<>();
userAges.put(1001, 25);
userAges.put(1002, 30);
```

### 示例4：两数之和（经典：值 -> 索引）

```java
import java.util.HashMap;
import java.util.Map;

public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();

    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }
        map.put(nums[i], i);
    }
    return new int[0];
}
```

## 重要概念澄清

### 1) HashMap 没有“索引”概念

- 数组：有索引（`arr[0]`, `arr[1]` ...）
- HashMap：只有键值对，通过 Key 查 Value（`map.get(key)`）

### 2) 键和值代表什么，由你决定

```java
import java.util.HashMap;
import java.util.Map;

Map<Integer, Integer> map = new HashMap<>();

// 数字 -> 出现次数
map.put(5, 3);

// 索引 -> 数值
map.put(0, 100);

// ID -> 分数
map.put(101, 95);
```

### 3) 为什么它“快”（内部结构简述）

HashMap 底层可以理解为：数组 + 链表/红黑树（解决哈希冲突）。

```java
// 简化版结构（仅用于理解）
Node<K, V>[] table;

static class Node<K, V> {
    final int hash;
    final K key;
    V value;
    Node<K, V> next;
}
```

补充几个经常考到/容易踩坑的点：

- **`hashCode()` 和 `equals()` 必须配套**：`HashMap` 通过 `hashCode()` 定位桶，再通过 `equals()` 精确比对 Key。
- **Key 尽量不可变**：如果 Key 的字段参与了 `hashCode/equals`，放进 Map 后又修改字段，可能导致“取不出来”。
- **扩容与树化**：当桶内冲突过多且满足条件时，会从链表转换为红黑树以优化性能（JDK8+）。
- **允许 null**：`HashMap` 允许一个 `null` Key、允许多个 `null` Value。
- **无序**：`HashMap` 不保证遍历顺序。
- **线程不安全**：并发场景用 `ConcurrentHashMap`。

## 拓展

### 1) `for (int num : nums)` 含义

这是增强型 for 循环（for-each）。含义：对数组 `nums` 的每个元素，依次赋值给 `num`。

```java
int[] nums = {1, 2, 3, 4, 5};

for (int i = 0; i < nums.length; i++) {
    int num = nums[i];
    System.out.println(num);
}

for (int num : nums) {
    System.out.println(num);
}
```

### 2) Map 和 HashMap 的关系

- `Map<K, V>`：接口（规范）
- `HashMap<K, V>`：实现类（具体实现）

```java
import java.util.HashMap;
import java.util.Map;

Map<String, Integer> map1 = new HashMap<>();
HashMap<String, Integer> map2 = new HashMap<>();
```

一般推荐用接口类型声明（`Map`），减少耦合。

### 3) Map 家族常见实现（怎么选）

- `HashMap`：最常用；无序；允许 null；线程不安全
- `LinkedHashMap`：保持插入顺序/访问顺序
- `TreeMap`：按 Key 排序（红黑树）
- `ConcurrentHashMap`：高并发场景首选
- `Hashtable`：历史产物；线程安全但不允许 null，通常不推荐新项目使用

## 常用方法速查

### 数组常用方法

```java
import java.util.Arrays;

int[] arr = new int[5];

arr.length;
Arrays.sort(arr);
Arrays.toString(arr);
Arrays.copyOf(arr, 10);
Arrays.fill(arr, 0);
Arrays.binarySearch(arr, 5);
```

### Map 接口核心方法

```java
import java.util.HashMap;
import java.util.Map;

Map<String, Integer> map = new HashMap<>();

map.put("key", 100);
map.get("key");
map.remove("key");
map.containsKey("key");
map.containsValue(100);

map.putAll(Map.of("a", 1));
map.clear();

map.keySet();
map.values();
map.entrySet();

map.size();
map.isEmpty();

map.getOrDefault("key", 0);
map.putIfAbsent("key", 200);
map.forEach((k, v) -> System.out.println(k + "=" + v));
```

### HashMap（类）相对 Map 接口的额外点

`HashMap` 作为具体类额外提供了如 `clone()` 之类的方法：

```java
import java.util.HashMap;

HashMap<String, Integer> hashMap = new HashMap<>();
hashMap.clone();
```

## 选择指南

| 场景 | 推荐实现 | 理由 |
| --- | --- | --- |
| 一般用途 | `HashMap` | 性能好、最常用 |
| 需要保持顺序 | `LinkedHashMap` | 保持插入/访问顺序 |
| 需要排序 | `TreeMap` | 自动按 Key 排序 |
| 高并发 | `ConcurrentHashMap` | 线程安全且性能好 |

## 总结

- `Map<Integer, Integer>` 存储的是“两个 Integer 的映射关系”，不是固定的“索引和值”。
- `HashMap` 通过 `hashCode/equals` 查找 Key；Key 尽量不可变。
- 根据需求选择实现：无序用 `HashMap`，要顺序用 `LinkedHashMap`，要排序用 `TreeMap`，并发用 `ConcurrentHashMap`。
