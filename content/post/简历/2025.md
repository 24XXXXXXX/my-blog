## 四、项目经历

### 1. 快食尚智慧点餐系统（个人 / 实战项目）

- **项目简介**：  
  快食尚是一个面向餐饮门店的线上点餐系统，包含：
  - C 端顾客使用的 **微信 / 支付宝小程序**（基于 uni-app 开发），支持扫码点餐、多端同步购物车、在线支付；  
  - B 端门店员工使用的 **网页后台管理系统**（基于 Vue 3 + Vite + TypeScript + Element Plus），用于菜品、套餐、订单、员工和数据统计管理；  
  - **后端服务** 使用 Java + Spring Boot + MyBatis 构建，提供完整的订单、菜品、用户、支付等接口，并对接 MySQL、Redis、阿里云 OSS 以及微信/支付宝支付。  

- **项目地址**：  
  - 后台管理临时访问地址：`https://4a40d4d3.r28.cpolar.top/`（通过 cpolar 暴露本地服务）  

- **主要职责与工作内容**：  

  1）**小程序端（uni-app 多端点餐）**  
  - 使用 **uni-app + Vue** 搭建小程序项目结构，配置 `pages.json`、`manifest.json` 等核心文件  
  - 完成 **扫码点餐流程**：扫描桌台二维码 → 获取桌台信息与当前状态 → 拉取菜品分类与菜单 → 加入购物车 → 创建订单 → 调用支付  
  - 使用 **WebSocket** 实现同桌多用户点餐时的购物车实时同步，提升多人点餐体验  
  - 封装通用的请求工具，统一处理请求前缀、Token、错误提示等逻辑  
  - 配合微信开发者工具完成真机调试与小程序配置（域名白名单、支付调试、权限配置等）  

  2）**网页后台管理端（Vue 3 + Vite + Element Plus）**  
  - 在 Vue 3 + TypeScript + Vite 项目中，基于 **Element Plus** 搭建后台基础布局（侧边菜单、顶栏、面包屑等）  
  - 使用 **Pinia + Vue Router 4** 实现登录鉴权、路由守卫与菜单权限控制  
  - 基于 **Axios** 封装请求层与拦截器，统一处理 Token 过期、错误码和全局 Loading / Message 提示  
  - 使用 **ECharts** 实现营收、订单量、用户数等数据的图表可视化（日报 / 周报 / 月报）  
  - 调用后端订单接口，完成订单列表、接单、派送、完成、取消等完整业务流程  
  - 集成 **WebSocket 实时通知** 与音频提醒，对新订单和催单进行实时推送  
  - 对样式使用 **SCSS** 管理全局主题变量和混入，提升样式复用性与可维护性  
  - 在 Vite 配置中使用别名、自动导入、SVG 图标插件，优化开发体验与构建性能  

  3）**后端服务（Spring Boot + MyBatis）**  
  - 基于 **Java 17 + Spring Boot 2.7**，拆分模块（`sky-common`、`sky-pojo`、`sky-server`），按职责划分实体与业务  
  - 使用 **MyBatis** 为菜品、套餐、分类、订单、用户、员工等模块编写 Mapper 与 Service 层逻辑  
  - 使用 **MySQL** 存储核心业务数据，按业务抽象数据表结构，在 `sky.sql` 中管理数据库脚本  
  - 集成 **Redis** 缓存，提高高频读写场景（如菜品、验证码、会话信息）的性能  
  - 使用 **JWT** 作为登录认证方案，实现后台登录与角色权限校验  
  - 通过 **Knife4j** 维护接口文档，方便前后端联调与测试  
  - 使用 **阿里云 OSS** 存储菜品图片等资源，并在后台管理端支持图片上传与展示  
  - 对接 **微信支付 / 支付宝支付 SDK**，完成下单、支付、回调通知与订单状态流转  
  - 使用 **PageHelper** 为后台列表提供通用分页能力，配合 **POI** 实现部分数据的 Excel 导出  
  - 结合 **Docker / docker-compose / Nginx** 等，完成后端与前端项目的容器化部署与反向代理配置
